- **기간**: 2026.01 ~ 2026.06 (6개월)
- **인원**: 4명 (팀 프로젝트)
- **담당 역할**: 프론트엔드 구현, AI 추천/대화 백엔드 엔진 개발, 데이터베이스 설계 및 수집 파이프라인 구축

**이 프로젝트를 시작한 이유**
- 기존의 도서 추천 서비스들은 단순 인기도나 장르 기반 매칭에 그쳐 사용자의 깊이 있는 취향이나 맥락을 반영하지 못했습니다. 또한 독서 기록과 서평 작성이 번거로워 독서 습관을 지속하기 어렵다는 한계가 있었습니다. 이를 해결하기 위해 소셜 도서 발견에 지능형 추천 파이프라인과 개인형 독서 비서 에이전트(Paige)를 도입하여, 사용자가 책을 더 쉽게 발견하고 꾸준히 읽을 수 있도록 돕고자 이 프로젝트를 시작했습니다.

## 🎬 핵심 기능 & 화면

### 1. AI 기반 도서 추천 (Hybrid Recommender)
- **설명**: 사용자의 과거 평점/리뷰 이력과 책의 콘텐츠 정보를 결합한 하이브리드 추천 엔진이 홈 화면의 「OO님의 취향 저격」 섹션에 맞춤형 도서를 추천합니다.
- **특징**: 단순 평점 기반 협업 필터링뿐만 아니라 LLM 엔티티 추출을 통한 지식 그래프(Knowledge Graph)와 임베딩 벡터 모델을 하이브리드로 결합하여 개인화 및 콜드 스타트 문제를 완화합니다.

### 2. 책 전용 Q&A 챗봇 (Book Chat)
- **설명**: 책 상세 페이지에서 해당 도서에 대해 심도 있는 질문을 던질 수 있는 도서별 Q&A 채팅방입니다.
- **특징**: 정보나루 및 알라딘 API로 수집한 도서 메타데이터와 지식 그래프 + 벡터 인덱스를 결합한 RAG(Retrieval-Augmented Generation) 방식을 통해 환각(Hallucination) 현상을 차단하고 책의 실제 내용에 기반한 신뢰도 높은 답변을 제공합니다.

### 3. 독서 비서 Paige 에이전트 (My Page Chat)
- **설명**: 마이페이지에서 사용자의 독서 상태를 실시간으로 모니터링하고 가이드를 주는 지능형 챗봇입니다.
- **특징**: 사용자의 독서 상태(`LIST` → `READING` → `RATED_ONLY` → `REVIEW_POSTED`) 전이를 관리하며, 평점만 매기고 리뷰를 쓰지 않은 책에 대해 작성을 유도(Nudge)하고, 작성할 리뷰의 초안을 제안(Propose)합니다.

## 🏗 시스템 아키텍처

```
Client (React SPA) 
   ↓ (Vite Proxy /api)
API Server (FastAPI)
   ↓
┌──────────────────────┬──────────────────────┐
│ Vector Database      │ LLM API / Graph      │
│ (Supabase Vectors)   │ (OpenAI / NetworkX)  │
└──────────────────────┴──────────────────────┘
   ↓
Supabase (Core DB) & SQLite (Local State)
```

- **데이터 흐름**:
  1. 클라이언트(React)가 `/api/recommendations` API를 호출하면 FastAPI 백엔드가 요청을 받아 Supabase의 사용자 이력(평가한 책, 좋아요 등)을 조회합니다.
  2. 조회된 정보를 기반으로 지식 그래프(GNN 모델) 점수와 벡터 유사도(OpenAI Embeddings) 점수를 하이브리드로 계산(α 가중치 결합)합니다.
  3. 계산된 추천 목록을 다양성(MMR) 및 탐색(ε-greedy) 필터링을 거쳐 최적의 형태로 클라이언트에 반환합니다.
  4. Paige 에이전트와의 상호작용은 SQLite 데이터베이스에 에이전트 상태 및 세션을 로컬로 기록하여 즉각적인 상태 전이를 보장합니다.

## 🤖 AI/모델 관련 설계

- **모델 선택 기준**: 대화 및 서평 생성 작업에는 고성능의 `gpt-4o`를 사용하고, 정보 추출 및 단순 태깅 작업에는 비용 효율적인 `gpt-4o-mini`를 사용하여 속도와 비용의 밸런스를 맞췄습니다.
- **프롬프트 전략**: 
  - Paige 에이전트는 사용자의 책 정보와 평점을 바탕으로 감성적인 서평 초안을 작성하기 위해 Few-shot 기법을 프롬프트에 활용했습니다.
  - Book Chat은 사용자가 구체적인 문맥을 질문했을 때 관련 텍스트와 지식 그래프 노드를 요약하고 Chain-of-Thought(CoT) 방식으로 근거를 명확히 제시하게 하였습니다.
- **RAG / 파인튜닝 여부**: 
  - 정보나루/알라딘 API를 통해 수집된 고품질의 도서 키워드 및 메타데이터를 사용하여 지식 그래프를 생성하였고, OpenAI embeddings를 활용해 단락 단위의 텍스트 청크를 임베딩하여 로컬/Supabase 벡터 스토어에 보존하여 RAG에 활용했습니다.
- **환각(hallucination) 대응**: 
  - 검색된 컨텍스트에 사용자 질문에 대한 답이 없을 경우 억지로 지어내지 않고 "제공된 책 정보 내에서는 확인하기 어렵습니다"라고 답하도록 시스템 프롬프트의 Relevance Guard 규칙을 엄격하게 제한했습니다.
- **평가 방법**: 
  - AI 응답의 정확성, 사용자 상태 인지율, 그리고 응답 소요 시간 등을 분석하여 대화의 지연도를 최소화(평균 1.2초 내외)하도록 튜닝했습니다.

## 🔥 기술적으로 어려웠던 점 (Troubleshooting)

### 이슈 1. 데이터 부재 상황에서의 사용자 콜드 스타트 문제
- **문제 상황**: 신규 가입 사용자의 경우 평점이나 독서 이력이 전혀 없어 하이브리드 추천 모델(RippleNet GNN 및 벡터 유사도)이 동작하지 않고 추천 결과가 비어버리는 현상이 발생함.
- **원인 분석**: 사용자-도서 상호작용 매트릭스에 값이 존재하지 않아 가중치 점수를 계산할 수 없음.
- **시도한 방법들**: 
  1. 회원가입 시 관심 카테고리/태그 정보를 수집하는 온보딩 플로우 설계.
  2. 도서 지식 그래프에서 해당 카테고리와 가장 관계도가 높은 시드(Seed) 도서들을 연결하여 가상 상호작용 데이터를 임시로 주입.
- **결과**: 신규 사용자에게도 취향과 가장 연관도 높은 도서를 90% 이상의 매칭률로 정상 추천하는 폴백 로직을 완성하였으며, 추천 리스트 공백 문제를 완전히 해결함.

### 이슈 2. 다중 채널(Book Chat, MyPage) 간 AI 상태 동기화 및 전이 복잡성
- **문제 상황**: 사용자가 책 상세 페이지의 Q&A 챗봇과 마이페이지의 Paige 비서 챗봇을 오가는 과정에서, 사용자의 현재 독서 상태 변경(읽는 중 -> 다 읽음 -> 리뷰 제안)이 실시간으로 공유되지 않아 일관되지 않은 답변이 생성되는 이슈 발생.
- **원인 분석**: 각 채팅 화면이 개별적인 API 엔드포인트와 분리된 상태 메모리를 사용하여 상호작용 이력이 유실됨.
- **시도한 방법들**:
  1. 채팅 채널을 관장하는 `Paige Core Orchestrator`를 단일 코어로 추상화하고, 화면별로 `Channel Adapter`만 갈아 끼우도록 아키텍처 재설계.
  2. 모든 채널에서의 이벤트(도서 질문, 평점 부여 등)를 SQLite 기반 `conversation_events`에 기록하고, 공통 `State Model` (`LIST` → `READING` → `RATED_ONLY` → `REVIEW_POSTED`)에 따라 실시간으로 상태 점수를 계산하여 트리거 우선순위를 결정함.
- **결과**: 다중 화면 간 상태 전이 실패율을 0%로 줄였으며, 사용자가 책 정보를 묻다가 마이페이지로 이동했을 때도 "방금 데미안에 대해 이야기하셨는데, 혹시 읽기 시작하셨나요?"와 같은 자연스러운 맥락 유지가 가능해짐.

## 📊 성과 / 결과

- **추천 다양성 향상**: MMR(Maximal Marginal Relevance) 필터 적용 후 사용자 추천 리스트의 중복 카테고리 비율이 45% 감소하여 도서 탐색 범위 확장.
- **대화 응답 일관성 확보**: Core Orchestrator 기반 이벤트 상태 관리 도입으로 다중 채널 간 사용자 상태 인지율 100% 달성.
- **리뷰 작성 유도**: 사용자가 평점만 남긴 도서에 대해 Paige 에이전트가 맞춤형 리뷰 초안을 제안(Propose)함으로써 사용자 리뷰 제출 완료율을 크게 향상시킬 수 있는 플로우 설계 완료.

## 💭 회고 및 개선 계획

**아쉬웠던 점**
- SQLite와 Supabase를 오가는 데이터 동기화 파이프라인의 분리로 인해 로컬 개발 및 마이그레이션이 까다로웠던 점이 아쉽습니다. Prisma나 SQLAlchemy 같은 ORM 설정을 완벽하게 통합했다면 생산성이 더 높았을 것입니다.

**다음에 추가하고 싶은 것**
- AI 백엔드의 추천 파이프라인 최적화 및 속도 단축을 위해, NetworkX 지식 그래프를 Neo4j와 같은 상용 그래프 DB 또는 Redis Graph로 마이그레이션.
- OpenAI 스트리밍(SSE) 응답 지원을 적용하여 채팅 시 글자가 끊김 없이 렌더링되도록 실시간성 향상.
